# Support

## Before Opening A Request

- Read [README.md](README.md) for project overview and quick start.
- Read [docs/getting-started.md](docs/getting-started.md) for setup.
- Check [docs/README.md](docs/README.md) for architecture, operations, and
  troubleshooting references.

## How To Get Help

- Usage questions and configuration help:
  open an issue using the Feature Request template and describe your scenario.
- Bug reports:
  open an issue using the Bug Report template with reproduction steps.
- Security issues:
  follow [SECURITY.md](SECURITY.md) and use private vulnerability reporting.

## Response Expectations

This project is maintained on a best-effort basis.

- Critical security reports: prioritized.
- Reproducible bugs: reviewed as capacity allows.
- General usage questions: best-effort support.

## What Maintainers Need

To help us respond quickly, include:

- Environment details (OS, Python version, deployment target).
- Exact commands and configuration used.
- Logs, stack traces, and relevant file paths.
- What you expected to happen and what happened instead.
