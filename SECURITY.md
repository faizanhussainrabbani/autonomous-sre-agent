# Security Policy

## Supported Versions

Security fixes are applied on a best-effort basis to the actively maintained
branch:

- `main`: supported

## Reporting A Vulnerability

Do not open public issues for security vulnerabilities.

Use GitHub private vulnerability reporting:

1. Open the repository Security tab.
2. Select "Report a vulnerability".
3. Provide reproduction details, impact, and affected components.

If private reporting is unavailable, contact @faizanhussain directly on GitHub
and include "SECURITY" in the message title.

## What To Include In Reports

- A clear description of the vulnerability.
- Steps to reproduce and expected vs actual behavior.
- Impact and potential blast radius.
- Relevant logs, commits, and environment context.

## Response Targets

- Initial acknowledgment: within 2 business days.
- Triage decision: within 5 business days.
- Remediation timeline: communicated after triage.

## Disclosure Policy

Please allow time for validation and remediation before public disclosure.
Once a fix is available, maintainers will coordinate responsible disclosure
details.
